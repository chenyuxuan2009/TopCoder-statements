Problem Statement for "SequenceSerializer"

Problem Statement

When transmitting data over a network, the data needs to be serialized into a sequence of bits - zeros and ones. Consider the task of serializing a sequence of integers. The simplest way to do this is to transmit a fixed number of bits per integer. In this problem, however, we will consider another method. First, we will choose some sequence of bits, called s. Next, we will convert each integer into a binary string with a length that is a positive multiple of the length of s (padding the front with zeros if neccessary). Thus, each integer can be considered as a number of groups of binary digits, where each group has a number of bits equal to the length of s. If s is a prefix of any of the binary strings, then our choice of s is invalid (as it may lead to an ambiguous decoding), and we can not use that s.

To transmit the data, we will transmit the groups of the first number, in order from left to right (most significant to least significant). If any of the groups are exactly equal to s, we will transmit those groups twice. After the entire first number is transmitted is this way, we will transmit s once, and then proceed with the next number in the same way. We do not need to transmit s after the last number is sent.

For example, consider transmitting the sequence of numbers {1,3,6,9} with s = "01". First, we convert each number to binary, and pad the left with zeros so that the lengths are each divisible by 2:
{01,11,0110,1001}
Now, when we transmit the data, we have to transmit each group that is equal to s twice: 
Number    :    1   |    |  3 |    |    6     |    |    9
Bits sent :  01 01 | 01 | 11 | 01 | 01 01 10 | 01 | 10 01 01
One way to think about this is that s indicates that a new number is about to start, unless it occurs twice in a row.

Different choices of s lead to a different number of bits being sent. Your task is, given a sequence of integers, sequence, find the value of s that requires the fewest bits be transmitted. Return the number of bits that need to be sent for this s.
