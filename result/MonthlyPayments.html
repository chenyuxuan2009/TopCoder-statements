Problem Statement for "MonthlyPayments"

Problem Statement

Given a JavaBean component, you are required to calculate monthly payments for a certain period.

Given class PaymentProcessor:

This class is a JavaBean component.

Mutator and accessor methods for the following properties are defined according to the JavaBean naming conventions, but are not shown here.

private int januaryProfile, februaryProfile, marchProfile, aprilProfile,
mayProfile, juneProfile, julyProfile, augustProfile,
septemberProfile, octoberProfile, novemberProfile, decemberProfile;

class PaymentProcessor implements Serializable {
  /**
   * Gets a country code that you can use to construct a Locale object.
   *
   * @return a lowercase two-letter ISO-639 country code 
   */
   public String getPaymentLanguage();

  /**
   * Expresses currency values in the locale corresponding to the
   * country code returned by getPaymentLanguage().
   * 
   * Each string is a currency expression with an integer value
   * between 0 and 1000000 (one million), inclusive.
   *
   * @return A collection of strings expressing currency values containing
   *         between 3 and 100 values, inclusive.  Contains no duplicates.
   */
  public Collection&lt;String&gt; getPaymentCollection();
}

DEFINITION

You must define a public class MonthlyPayments with public method int[] getValues(int monthNumber, int profile, int minimumPaymentValue).

You must implement getValues as follows:

SET THE PROFILES

Instantiate PaymentProcessor with the default constructor and use the resulting bean to calculate monthly payments.

Set the profile for the month corresponding to monthNumber and all months preceding it in the calendar. For example, if monthNumber is 2, set the profile for February and the preceding month January.

The names of the months in calendar order are: January, February, March, April, May, June, July, August, September, October, November, December.

CALCULATE MONTHLY PAYMENTS

Now call getPaymentLanguage() and use the result to construct a Locale paymentLocale. Pass paymentLocale to NumberFormat.getCurrencyInstance(Locale) to obtain a currency format. Next, call getPaymentCollection() to obtain a collection of strings. Parse each string with the currency format and get the integer value of the resulting Number.

Return an int[] containing the payment values that are greater than or equal to minimumPaymentValue, in the order in which the collection iterates.

Constraints

  • monthNumber will be between 1 and 12, inclusive.
  • profile will be between 1 and 8192, inclusive.
  • minimumPaymentValue will be between 0 and 1000000 (one million), inclusive.
