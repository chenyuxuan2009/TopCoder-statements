Problem Statement for "Clipping"

Problem Statement

The examples contain images.

An important part of 3-D rendering is polygon clipping. This tells you which parts of various objects can be seen from a certain angle. In this problem, we'll be dealing with a similar, but much simpler task - 2-D line segment clipping. Consider a viewpoint from the origin, looking at the upper right quadrant of the Cartesian coordinate space (+x, +y). Assuming that the field of view encompasses the full 90 degrees, your task is to determine which of a number of line segments contribute to the view. More formally, a line segment contributes to the view if there are an infinite number of points (with real coordinates) on that line segment such that the line segment between the origin and each of those points does not intersect any segments in the space.

You will be given a String[], segs, each element of which represents a line segment from (X1,Y1) to (X2,Y2) in the form "X1 Y1 X2 Y2". Return the number of the segments which contribute to the view.

Definition

Class:
Clipping
Method:
clip
Parameters:
String[]
Returns:
int
Method signature:
int clip(String[] segs)
(be sure your method is public)

Constraints

  • segs will contain between 1 and 50 elements, inclusive.
  • Each element of segs will be formatted as "X1 Y1 X2 Y2".
  • Each coordinate will be an integer between 0 and 100, inclusive, and will not contain any extraneous leading zeros.
  • No two segments will intersect at more than a single point.
  • No segment will touch (0,0).

Examples

  1. {"10 0 0 10","5 0 0 5"}

    Returns: 1

    The segment from (10,0) to (0,10) is completely clipped by the segment from (5,0) to (0,5)

  2. {"10 0 5 10","10 5 0 10","10 0 0 10"}

    Returns: 1

  3. {"10 0 5 10","10 5 0 10","9 0 0 9"}

    Returns: 1

  4. {"10 0 5 10","10 5 0 10","11 0 0 11"}

    Returns: 3

  5. {"10 0 5 5","5 5 0 10","2 1 8 4","6 5 5 6"}

    Returns: 2

    The segment from (2,1) to (8,4) is parallel to the line of sight from the origin, and hence only a single point can be seen, so it doesn't contribute.

  6. {"3 6 0 9","9 1 7 0","5 5 7 7"}

    Returns: 2

  7. {"4 7 2 0","10 1 10 6","0 10 9 0"}

    Returns: 2

  8. {"47 36 58 49","6 98 54 0","2 42 2 96"}

    Returns: 2

  9. {"2 10 2 3","6 4 7 1","4 1 1 9"}

    Returns: 3

  10. {"9 8 8 3","4 9 1 3","5 10 5 1"}

    Returns: 2

  11. { "41 83 52 50", "51 54 52 48", "51 55 52 46", "51 56 52 44", "51 57 52 42", "51 58 52 40", "51 59 52 38", "51 60 52 36", "51 61 52 34", "51 62 52 32", "51 63 52 30", "51 64 52 28", "51 65 52 26", "51 66 52 24", "51 67 52 22", "51 68 52 20", "51 69 52 18", "51 70 52 16", "51 71 52 14", "51 72 52 12", "51 73 52 10", "51 74 52 8", "51 75 52 6" }

    Returns: 2

    All lines intersect at point (51 1/3, 52), so only 2 are visible.

  12. { "51 0 49 100", "33 50 67 49", "1 51 99 48", "0 50 100 50" }

    Returns: 4

  13. { "1 0 95 96", "0 1 97 96", "0 94 99 0" }

    Returns: 3

    Only a very short portion of segment 2 is visible (less than 0.001 in length).

  14. { "61 2 10 61", "24 71 9 59", "7 51 21 87" }

    Returns: 2

    Only .000414 of segment 1 is visible.
