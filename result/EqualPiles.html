Problem Statement for "EqualPiles"

Problem Statement

It's Halloween. You have N bags of candy to hand out to trick-or-treating kids. The bags are numbered from 0 to N-1, inclusive.

Each bag contains some number of candies: bag number i contains (1 + (i*i modulo M)) candies.


It's best if all bags of candy you hand out are equally large - that way siblings won't argue that one of them got more than the other.

You have T minutes before the kids arrive. During each minute you can open one bag of candies, eat some of the candies in the bag (possibly all of them) and re-seal the bag.

You want to produce as many equal bags of candy as possible. (The number of candies in those bags does not matter.)

Calculate and return the largest number of equal bags you can obtain in T minutes.

Definition

Class:
EqualPiles
Method:
equalize
Parameters:
int, int, int
Returns:
int
Method signature:
int equalize(int N, int M, int T)
(be sure your method is public)

Notes

  • The formula for the number of candies in each bag is just to keep the input size small. The reference solution would work for any numbers of candies in bags.
  • Watch out for integer overflow when computing the initial number of candies in each bag. In particular, the value i*i will not always fit into a 32-bit integer.

Constraints

  • N will be between 1 and 300,000, inclusive.
  • M will be between 1 and 10^9, inclusive.
  • T will be between 0 and N, inclusive.
