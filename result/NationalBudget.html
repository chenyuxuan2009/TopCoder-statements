Problem Statement for "NationalBudget"

Problem Statement

Preparing a national budget involves sums of large numbers. These numbers should be formatted in a nice way. Your task is, given some nicely formatted costs (in US dollars and cents), add up the costs and return the nicely formatted total cost.

The nice format is:
Numbers are decimal (base ten).
Always display the cents as two digits to the right of the decimal point, even if this gives trailing zeros.
Always display at least one digit to the left of the decimal point.
Unless the absolute value of the number is less than one dollar, use no leading zeros.
If the absolute value of the number is less than one dollar there should be one zero to the left of the decimal point.
If the number of digits to the left of the decimal point is five or more, then insert commas between groups of three digits from right to left from the decimal point. (see examples)
If the number is positive (greater than or equal to zero) the first character is &quot;$&quot;.
If the number is negative (strictly less than zero) then the first two characters are &quot;-$&quot;.

Definition

Class:
NationalBudget
Method:
tally
Parameters:
String[]
Returns:
String
Method signature:
String tally(String[] costs)
(be sure your method is public)

Constraints

  • costs will have between 1 and 50 elements inclusive.
  • Each element of costs will be formatted as described above.
  • Each element of costs will have between 5 and 20 characters inclusive.
  • Each character will be from the set "-$0123456789.," (quotes for clarity) no other characters will be present.
  • Each element of costs will be between "-$100,000,000,000.00" and "$100,000,000,000.00" inclusive.
