Problem Statement for "HexGridPaths"

Problem Statement

Consider an infinite hexagonal grid numbered in a spiral (as shown in http://www.cut-the-knot.org/ctk/HexMosaic.gif , for example).

You are on the cell start. In each step, you can move from your current cell into one of the six adjacent ones. You must get to the cell finish using the smallest possible number of steps. Depending on the locations of your start and finish, there may be many possible paths. We will call them the good paths.

Some of the cells contain obstacles, such as mud, thorns, etc. The numbers of these cells are given as int[] obstacles. Going through these cells is unpleasant and you want to avoid all such cells.

Let P be the number of good paths that avoid all obstacles. Compute the value (P modulo 1,000,000,009).

Definition

Class:
HexGridPaths
Method:
pathCount
Parameters:
long, long, long[]
Returns:
int
Method signature:
int pathCount(long start, long finish, long[] obstacles)
(be sure your method is public)

Notes

  • The smallest possible number of steps in the definition of a good path is calculated for the empty grid, i.e., assuming that there are no obstacles. See example 3 for clarification.

Constraints

  • start will be between 1 and 10^12, inclusive.
  • finish will be between 1 and 10^12, inclusive.
  • start will not be equal to finish
  • obstacles will contain between 0 and 50 elements, inclusive.
  • Each element of obstacles will be between 1 and 10^12, inclusive.
  • No two elements of obstacles will be equal.
  • No element of obstacles will be equal to start.
  • No element of obstacles will be equal to finish.
