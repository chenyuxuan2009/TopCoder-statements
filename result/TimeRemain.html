Problem Statement for "TimeRemain"

Problem Statement

Little Lucy loves to download music from the Internet. Recently she discovered a very frustrating bug in her downloading software. When Lucy downloads some files the total bandwidth is distributed evenly between all files. Currently, the download agent estimates the remaining time for each download as remaining_size / current_speed. This approach is bad since it does not take into account the possibility of some downloads finishing earlier than others, thus freeing up additional bandwidth for remaining downloads.

You are given the total bandwidth (in bytes per second) and a int[] fileSizes, the i-th element of which is the size of the i-th file (in bytes). Return a double[] containing the actual times when the downloads will finish (in seconds). The i-th element of the return value should correspond to the i-th file.

Definition

Class:
TimeRemain
Method:
estimateRemainingTime
Parameters:
int[], int
Returns:
double[]
Method signature:
double[] estimateRemainingTime(int[] fileSizes, int bandwidth)
(be sure your method is public)

Notes

  • Each element of the return value must be accurate to within an absolute or relative value of 1e-9.

Constraints

  • fileSizes will contain between 1 and 50 elements, inclusive.
  • bandwidth and each element of fileSizes will be between 1 and 1000000000, inclusive.
