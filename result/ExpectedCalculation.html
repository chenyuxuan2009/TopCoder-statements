Problem Statement for "ExpectedCalculation"

Problem Statement

You have written a small piece of code that will help you finish your long homework assignment. Before actually using it on your homework, you want to test your program's speed. You will be given a int[] times containing running times in seconds. You will also get a int[] frequencies. Element i of frequencies tells you how many tests took times[i] seconds each. For example,
times        =  {10,2,5}
frequencies  =  {3,1,4}
means that 3 tests took 10 seconds, 1 test took 2 seconds, and 4 tests took 5 seconds. Return the average number of seconds taken by all of the tests. Here the average would be: 
	(5+5+5+5+2+10+10+10)/(3+1+4) = 52/8 = 6.5

Definition

Class:
ExpectedCalculation
Method:
getExpectation
Parameters:
int[], int[]
Returns:
double
Method signature:
double getExpectation(int[] times, int[] frequencies)
(be sure your method is public)

Notes

  • Solutions must be accurate to 1e-9 absolute or relative. In other words, standard double arithmetic provides enough precision.

Constraints

  • times will contain between 2 and 50 elements inclusive.
  • frequencies will contain the same number of elements as times.
  • Each element of times will be between 1 and 1000 inclusive.
  • Each element of frequencies will be between 1 and 1000 inclusive.
  • times will not contain any duplicate values.
