Problem Statement for "AmountToPay"

Problem Statement

You must write a program which will process bills at the store. You will be given two String[]s, goods and bill, containing information about goods and a bill. Each element of goods will be of the form "ID COST AMOUNT", where ID, COST and AMOUNT are positive integers without leading zeroes. ID is a unique code for the goods, AMOUNT specifies the number of units in the package, and COST specifies the cost of this package. Each element of bill will be formatted as "ID AMOUNT", where ID is the unique code of the goods sold, and AMOUNT is the number of units sold. Your program should compute the total amount for the bill.
Note that, while the goods are priced in packages, customers need not buy full packages. Hence, if a package of 4 units costs 8, then 2 units costs 4, and 6 units costs 12.

Definition

Class:
AmountToPay
Method:
calculate
Parameters:
String[], String[]
Returns:
int
Method signature:
int calculate(String[] goods, String[] bill)
(be sure your method is public)

Notes

  • bill may contain several elements with the same ID.

Constraints

  • goods and bill will each contain between 1 and 50 elements, inclusive.
  • Each element of goods will contain 3 space-delimited integers without any leading zeroes.
  • Each element of bill will contain 2 space-delimited integers without any leading zeroes.
  • Each ID in bill will appear as an ID in goods.
  • For each element of goods, the price for 1 unit of goods will be integral (COST will be always divisible by AMOUNT).
  • All numbers in all elements of goods and bill will be between 1 and 10000, inclusive.
  • All IDs in goods will be unique.
