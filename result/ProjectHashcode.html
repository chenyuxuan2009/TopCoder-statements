Problem Statement for "ProjectHashcode"

Problem Statement

You are given a project description. Calculate the hashcode for this project.

Given class ObjectT:

Represents any instance with a name.

class ObjectT {
  /**
   *  Returns the name of this ObjectT.
   *
   *  @return A String containing between 1 and 19 characters, inclusive.
   *          Each character in the string is one of 'a'-'z', 'A'-'Z', '0'-'9', '_'.
   */
  public String getName();
}

Given class ClassT that extends ObjectT:

Represents a single class in a project. Each class has a name that can be obtained with a call to the inherited ObjectT.getName() method.

Given class PackageT:

Represents a part of a project called &quot;package&quot;. Each package has a name that can be obtained with a call to the inherited ObjectT.getName() method. Packages might contain other packages and classes.

class PackageT extends ObjectT {
  /**
   *  Gets the list of all packages directly contained in this PackageT. 
   *  The order of packages in the list is not guaranteed.
   *
   *  @return A list with a maximum of 100 packages.
   */
  public List&lt;PackageT&gt; getPackages();

  /**
   *  Returns the list of all classes directly contained within this PackageT. 
   *  The order of the classes in the list is not guaranteed.
   *
   *  @return A list with a maximum of 100 classes.
   */
  public List&lt;ClassT&gt; getClasses();
}

Given class Project:

Represents a project. Projects are similar to packages except that they do not directly contain any classes. A single project can contain (directly or indirectly) a maximum of 100 packages, and (indirectly) a maximum of 100 classes.

There cannot be name conflicts within a project. If you consider an ObjectT[] with all classes and packages directly or indirectly contained within the Project, and the Project itself, then all items will have pairwise distinct names.

class Project extends ObjectT {
  /**
   *  Gets the list of all packages directly contained within this Project. 
   *  The order of packages in the list is not guaranteed.
   *
   *  @return A list with a maximum of 100 packages.
   */
  public List&lt;PackageT&gt; getPackages();
}

Given classes ProjectStorage and NoSuchProjectException:

Provides access to all the existing projects.

class ProjectStorage {
  /**
   * Gets the requested project instance.
   *
   * @param projectName The name of the project you want to access to.
   * @return The project named projectName.
   * @throws NoSuchProjectException if such project does not exist.
   */
  public static Project getProject(String projectName) throws NoSuchProjectException;
}

Given class HashCodeGenerator:

Generates hashcodes for projects.

class HashCodeGenerator {
  /**
   * Generates the hash integer for a project description.
   *
   * @param description The description string for a project.
   * @return The hashcode for a project with this description string.
   */
  public static int generateHash(String description);
}

DEFINITION

You must define public class ProjectHashcode with public method int getHashcode(String projectName). projectName contains between 1 and 19 characters, inclusive. Each character of projectName is a letter ('a' - 'z', 'A' - 'Z'), a digit ('0' - '9') or an underscore ('_').

To implement this method, calculate the description string for the project named projectName. Return HashcodeGenerator.generateHash(String) for the description string. If the project named projectName does not exist, return -1.

CLASS DESCRIPTION STRING

The description string for a class is simply the first character of its name.

PACKAGE DESCRIPTION STRING

To calculate the description string for a package, separately order all its direct packages and all its direct classes according to the following rules:

  • If package/class A and package/class B have names of different lengths, then the package/class with the shorter name should go first.
  • If package/class A and package/class B have names of the same length, then A should go before B if and only if name(A).compareTo(name(B)) &lt; 0, where name(A) and name(B) are the names of A and B, correspondingly.

Concatenate the description strings for all its direct packages (in the order described above), then add the description strings for all its direct classes (in the order described above) and finally add the first character of the package's name.

PROJECT DESCRIPTION STRING

To calculate the description string for a project, use the same rules and just treat it like a package without direct classes.
