Problem Statement for "TwistedNumbers"

Problem Statement

Let us call an integer x lucky if ends with 777. For example, numbers 2777, 23777, 777 are lucky, but numbers 317, 21377, 7 are not.

Let us call an integer x twisted if ax + b is lucky for some 1 &lt;= a &lt;= A, 0 &lt;= b &lt;= B.

You are given ints X, A and B. Return "YES" if X is twisted, or "NO" if it is not (all quotes for clarity).

Definition

Class:
TwistedNumbers
Method:
check
Parameters:
int, int, int
Returns:
String
Method signature:
String check(int X, int A, int B)
(be sure your method is public)

Constraints

  • X will be betweeen 1 and 1000, inclusive.
  • A will be between 1 and 10, inclusive.
  • B will be between 0 and 10, inclusive.
