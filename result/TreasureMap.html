Problem Statement for "TreasureMap"

Problem Statement

Robert has come into possession of an ancient treasure map. The map depicts a rectangular cave, and is divided into square cells representing the cave's chambers. The treasure is located in one of the chambers. Each of the remaining cells contains an arrow that is supposed to show the direction from that chamber to the treasure.

Robert will enter the cave in the upper left corner of the map. He intends to follow the arrows on the map and find the treasure. After looking at the arrows for a while, he got confused and is no longer sure that he will reach the treasure: it is possible that the arrows will lead him off the map, or have him going in circles. He wants to know if he will reach the treasure and, if yes, how many chambers he will go through before reaching it.

You will be given a String[] cave representing Robert's map. The treasure is marked with a period ('.'), while arrows are represented with characters '&lt;', '&gt;', '^' and 'v'. The chamber Robert first enters (the upper-left corner of the map) is character (0, 0) in cave. Return the number of chambers Robert goes through before finding the treasure. If he never reaches the treasure, return -1.

Constraints

  • cave will contain between 1 and 50 elements, inclusive.
  • Each element of cave will contain between 1 and 50 characters, inclusive.
  • Each element of cave will contain the same number of characters.
  • Each character in cave will be one of '.' (ASCII value 46), '<' (ASCII value 60), '>' (ASCII value 62), '^' (ASCII value 94) and 'v' (ASCII value 118).
  • There will be exactly one period character in cave.
