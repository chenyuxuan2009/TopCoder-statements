Problem Statement for "HockeySticks"

Problem Statement

Given an array A, an inversion is a pair of indices (i, j) such that i &lt; j but A[i] &gt; A[j].

A triple inversion is then a triple of indices (i, j, k) such that all three pairs (i, j), (j, k) and (i, k) are inversions.

For example, in the array A = {10, 20, 30, 80, 50, 60, 40, 70} the pair of indices (4, 6) is an inversion because 4 &lt; 6 while A[4] = 50 &gt; A[6] = 40. The pairs (3, 4) and (3, 6) are also inversions, which makes (3, 4, 6) a triple inversion.


A hockey stick is a 4-tuple of indices (i, j, k, l) such that (i, j, k) is a triple inversion, k &lt; l, and (k, l) is not an inversion.

In the array A from the previous example the 4-tuple of indices (3, 4, 6, 7) is an example of a hockey stick.


You are given an array A. (The exact input format is specified below.) Count all hockey sticks in A and return their count modulo (10^9 + 7).


In order to keep the input size small while still allowing meaningful challenges, most of the array A will be pseudorandom. You are given its length N, a modulus M, and the int[] Aprefix with a prefix of the array A. Please use the pseudocode below to generate the rest of A. Watch out for integer overflow: use 64-bit integers during the calculation where appropriate. 


L = length(Aprefix)

allocate an array A[0..N-1]

for i = 0 to L-1:
    A[i] = Aprefix[i]

for i = L to N-1:
    A[i] = (A[i-1] * 1103515245 + 12345) modulo 2^31

for i = L to N-1:
    A[i] = A[i] modulo M

  • Aprefix will have at most 100 elements.
  • Each element of Aprefix will be between 0 and M-1, inclusive.
