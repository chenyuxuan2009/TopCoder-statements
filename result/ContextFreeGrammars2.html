Problem Statement

A context free grammar is a set of words represented by a set of derivation rules, each written as
&lt;nonterminal&gt; ::= &lt;expression&gt;
where &lt;nonterminal&gt; is a symbol, and &lt;expression&gt; is a non-empty sequence of symbols (terminals and/or nonterminals) that represents a possible substitution for &lt;nonterminal&gt;. Symbols that never appear on the left side of a rule are called terminals.

If you have several rules for the same nonterminal:
&lt;nonterminal&gt; ::= &lt;expr0&gt;
&lt;nonterminal&gt; ::= &lt;expr1&gt;
...
&lt;nonterminal&gt; ::= &lt;exprn&gt;
it is also possible to write them in a single line, separated by pipes:
&lt;nonterminal&gt; ::= &lt;expr0&gt; | &lt;expr1&gt; | ... | &lt;exprn&gt;

In this problem we will use uppercase letters exclusively for nonterminals and lowercase letters exclusively for terminals. Also, for simplicity, all derivation rules for the same nonterminal will appear in a single line.

Given a specificacion as described above, the set of words that can be derived from a given nonterminal is the set of finite sequences of terminals (lowercase letters) that are the result of iteratively applying rules replacing a nonterminal for the right side of one of its rules. Note that words do not contain nonterminals (uppercase letters).

For example, this specification:
A ::= BD
B ::= bB | b | Bb
D ::= dD | d

we can derive from B the set of words that have one or more b's (b, bb, bbb, etc), from D the set of words that have one or more d's (d, dd, ddd, etc), and from A the set of words that have one or more b's followed by one or more d's (bd, bbd, bddd, bbbddd, etc.).

In this problem you are asked to evaluate the set of words that can be derived from each nonterminal.

You are given the set of rules as a String[], where each element is formatted as described above. Return a String[] with the same number of elements of rules, where element i represents the evaluation of the set of words that can be derived from the nonterminal whose rule is given in element i of rules. If the set of words is empty, the evaluation should be "EMPTY" (quotes for clarity), otherwise it should be "X - Y" (quotes for clarity) where X and Y should be replaced for the minimum and maximum length of a word in the set, respectively, without leading zeroes. If there is no maximum length word, replace Y with a single asterisk ('*'). If X and/or Y would be replaced by an integer greater than 1000000000 (10^9), replace it by "&gt;1000000000" (quotes for clarity) instead. See examples for further clarification.
