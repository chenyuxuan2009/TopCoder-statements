Problem Statement for "RandomRoadBuilding"

Problem Statement

Absurdistan is a country with several cities. Some pairs of cities are currently connected by bidirectional roads. Whenever a road is built in Absurdistan, the workers build bridges where necessary to make sure that no two roads intersect outside the cities. The current road network was built using this rule.

As one would expect, the road building process in Absurdistan is really absurd. Each year the Grand Visier randomly picks a pair of distinct cities, and then flips a biased coin. If the coin ends up heads, the workers will build a new road between the chosen two cities (even if multiple such roads exist already). If the coin ends up tails, the workers will destroy all roads that directly connect the chosen two cities.

The coin is biased in such a way that it is more likely that a new road will be built. Building a road or destroying the existing roads always takes the workers exactly one year. The entire process will terminate as soon as the road network connects all cities in the country.

You will be given a String[] countryMap containing the current map of Absurdistan, and a double buildProbability giving the probability that the biased coin ends up heads. The j-th character of the i-th element of countryMap is a digit representing the current number of roads connecting the i-th and the j-th city.

Return the expected number of years the process will take.

Definition

Class:
RandomRoadBuilding
Method:
expectedTime
Parameters:
String[], double
Returns:
double
Method signature:
double expectedTime(String[] countryMap, double buildProbability)
(be sure your method is public)

Notes

  • The returned value must have an absolute or relative error less than 1e-9.

Constraints

  • countryMap will contain between 1 and 7 elements, inclusive.
  • Each element in countryMap will contain the same number of characters as the number of elements in countryMap.
  • Each character in each element of countryMap will be a digit ('0'-'9').
  • The i-th character of the i-th element of countryMap will be '0' for all i.
  • The i-th character of the j-th element of countryMap will be equal to the j-th character of the i-th element for all pairs of i and j.
  • buildProbability will be between 0.51 and 1, inclusive.
