Problem Statement for "InverseMatrixTree"

Problem Statement

Here's one traditional graph theory task:

You are given a simple undirected graph G with n nodes. Please count the number of spanning trees, and return that number modulo 1,000,000,007.

The above task can be solved in O(n^3) time by using Kirchhoff's matrix tree theorem (https://en.wikipedia.org/wiki/Kirchhoff%27s_theorem).

This task is the inverse of the above task. You are given the output for the above problem: an int r. Your task is to construct any valid input that produces this output. More precisely, you have to construct a graph G with the following properties:
  • G is undirected.
  • G is simple. (Each pair of vertices is connected by at most one edge. There are no self-loops.)
  • The number of nodes in G is between 1 and 300, inclusive.
  • The number of edges in G is between 0 and 1000, inclusive.
  • (S modulo 1,000,000,007) = r, where S is the total number of spanning trees of G.
Construct any such graph G. Label its vertices 0 through n-1. Then, return a int[] with the description of your G. This int[] should have m+1 elements, where m is the number of edges in G. Element 0 of the return value should be n: the number of nodes. Each of the other elements encodes one edge of G: if G contains an edge (i,j), the return value should contain the number (i*n + j).

You may assume that each valid test case has a solution. If there are multiple solutions, you may return any of them.
