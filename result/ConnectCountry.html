Problem Statement for "ConnectCountry"

Problem Statement

In Absurdistan there are n towns, numbered from 1 to n. From time to time, the Grand Vizier of Absurdistan sends a new order for a road to be built between two given towns. Sadly, all the road workers in the country are illiterate. Thus they just randomly pick two different towns a and b and build a bidirectional road between them (even when those towns are already connected by several roads). The roads are built in such a manner that the only points where a traveler can enter or leave it are the towns it connects. The road administration has resources to build exactly one road per year.

You are given n and a list of roads that have already been built: the int[]s X and Y such that for each valid i there is a road that connects towns X[i] and Y[i]. Return the expected number of additional years it will take in order to connect all the cities in Absurdistan.

Definition

Class:
ConnectCountry
Method:
expectedYears
Parameters:
int, int[], int[]
Returns:
double
Method signature:
double expectedYears(int n, int[] X, int[] Y)
(be sure your method is public)

Notes

  • Your return value must have an absolute or a relative error at most 1e-9.
  • In the current country, as described by X and Y, there can already be pairs of towns that are connected by more than one direct road.

Constraints

  • n will be between 2 and 35, inclusive.
  • X will have between 0 and 500 elements, inclusive.
  • Y will have the same number of elements as X.
  • Each element of X will be between 1 and n, inclusive.
  • Each element of Y will be between 1 and n, inclusive.
  • For each valid i, X[i] will not be equal to Y[i].
