Problem Statement for "FairBandwidth"

Problem Statement

Some time ago, our company was smaller and we had less employees, so we shared the bandwidth equally among them, and if one of them was not using all his assigned bandwidth then the unused part was lost. However, in recent years, our number of employees grew a lot faster than our bandwidth, so there are now many employees with limited connectivity and others with unusued bandwidth, which is a poor use of a scarce resource.


For this reason, a new algorithm for bandwidth sharing must be implemented. Let's define an employee as satisfied if his assigned bandwidth is greater than or equal to his needed bandwidth. This new algorithm must be efficient and fair, which means:

  • If there is a satisfied user with unused bandwidth, there must be no unsatisfied users.
  • If a user is satisfied, all users that need less bandwidth than him must be satisfied too.
  • If a user is unsatisfied, then all users who need the same bandwidth as him or more must have the same assigned bandwidth as him.
  • Each unsatisfied user must have at least as much bandwidth as each satisfied user.

Create a method share that given the needed bandwidth by each user in a int[] needed and the total bandwidth available, returns a double[] with the same number of elements as needed, where the k-th element of the return value will be the assigned bandwidth to the k-th user.


Definition

Class:
FairBandwidth
Method:
share
Parameters:
int[], int
Returns:
double[]
Method signature:
double[] share(int[] needed, int bandwidth)
(be sure your method is public)

Notes

  • The returned values must be accurate to the correct results within a relative or absolute value of 1E-9.

Constraints

  • needed will contain between 1 and 50 elements, inclusive.
  • Each element of needed will be between 1 and 100, inclusive.
  • bandwidth will be between 1 and 5000, inclusive.
  • bandwidth will not exceed the sum of all elements of needed.
