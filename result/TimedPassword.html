Problem Statement for "TimedPassword"

Problem Statement

It is a common issue in big companies that computer users tend to use easy and simple passwords like "abcdef", common words like "house" or "tree" or simply their usernames. For that reason, companies tend to write policies (and enforce them through automated programs) that specify certain rules that must be followed when setting a new password.


Even with those measures, there are always some vulnerabilities in software that allow crackers to get passwords without using brute force, so we decided to patent a new way to discover if a password was written by the intended user. After a user types his password many times, he will start repeating the same timing pattern when typing the password. To verify if the typed password is correct, the system takes into account the following considerations:

  • If the user's input is not the same as the correct password, then the password is invalid.
  • If the timing error exceeds maxError then the password is invalid.
  • In any other case the password is considered valid and access is granted.

The timing error must be calculated only if the user's input is the same as the correct password. To calculate it, the system accumulates the error for each character typed after the first. The timing error of the k-th typed character is the absolute difference between inputTime[k]-inputTime[k-1] and correctTime[k]-correctTime[k-1].


Create a method valid that given the input password, the inputTime, the correct password, the correctTime and the maxError allowed, returns the String "ACCESS GRANTED" if the password is correct and the timing error does not exceed maxError, and returns "ACCESS DENIED" in any other case (all quotes for clarity).


Definition

Method:
valid
Parameters:
String, int[], String, int[], int
Returns:
String
Method signature:
String valid(String input, int[] inputTime, String correct, int[] correctTime, int maxError)
(be sure your method is public)

Constraints

  • input will contain between 1 and 50 characters, inclusive.
  • Each character of input will be a lowercase letter ('a'-'z').
  • inputTime will contain the same number of elements as the number of characters in input.
  • Each element of inputTime will be between 0 and 5000, inclusive.
  • All elements of inputTime will be strictly increasing.
  • The first element of inputTime will be 0.
  • correct will contain between 1 and 50 characters, inclusive.
  • Each character of correct will be an lowercase letter ('a'-'z').
  • correctTime will contain the same number of elements as the number of characters in correct.
  • Each element of correctTime will be between 0 and 5000, inclusive.
  • All elements of correctTime will be strictly increasing.
  • The first element of correctTime will be 0.
  • maxError will be between 0 and 50000, inclusive.
