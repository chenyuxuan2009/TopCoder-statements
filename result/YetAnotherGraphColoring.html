Problem Statement for "YetAnotherGraphColoring"

Problem Statement

There are n vertices labeled 1, 2, ..., n. Form a graph by adding an edge between each pair of vertices x, y such that x &lt; y and a[x] &gt; a[y].

Find and return the minimum number of colours needed to colour the graph. (Usual restriction applies: there can be no edge with its endpoints having the same colour.)


You are given the int n: the number of vertices. You are also given the ints a1, a2, x, y, z and m. Please use the following pseudocode to generate the array a[] used above. 


a[1] = a1
a[2] = a2
for i = 3 to n:
    a[i] = (x * a[i - 1] + y * a[i - 2] + z) modulo m

Constraints

  • 2 <= n <= 200,000
  • 0 <= a1, a2, x, y, z <= 10^9
  • 1 <= m <= 10^9
