Problem Statement for "TheLuckySequence"

Problem Statement

John thinks 4 and 7 are lucky digits, and all other digits are not lucky. A lucky number is a number that contains only lucky digits in decimal notation.

A lucky sequence is a sequence of length numbers A[0], A[1], ..., A[length - 1] that satisfies all of the following properties:

  • Each number A[i] is lucky, where 0 &lt;= i &lt; length.
  • For each i, where 0 &lt;= i &lt; length, there exists at least one j such that A[i] = numbers[j].
  • For each i, where 0 &lt;= i &lt; length - 1, the last digit of A[i] is the same as the first digit of A[i + 1].

You are given a int[] numbers and an int length. Return the number of distinct lucky sequences modulo 1234567891.

Definition

Class:
TheLuckySequence
Method:
count
Parameters:
int[], int
Returns:
int
Method signature:
int count(int[] numbers, int length)
(be sure your method is public)

Notes

  • Two lucky sequences A[0], A[1], ..., A[length - 1] and B[0], B[1], ..., B[length - 1] are considered distinct if there exists i, 0 &lt;= i &lt; length, such that A[i] &ne; B[i].

Constraints

  • numbers will contain between 1 and 50 elements, inclusive.
  • Each element of numbers will be between 1 and 1,000,000,000, inclusive.
  • length will be between 1 and 1,000,000,000, inclusive.
