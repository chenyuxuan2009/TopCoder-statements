Problem Statement for "StringMemory"

Problem Statement

In many languages strings are represented by immutable data structures. This means that the location where the string contents are stored in memory may not be altered. In some situations, this leads to a large number of similar strings being stored and wasting memory. In this problem, you will consider the situation where each String in s needs to be stored in memory. However, to save space, if one String is a substring of another one, you may forego storing the shorter String in memory. Instead, you may simply store the offset and length of the shorter String. For example, if given {"abcde","bcd"}, only the first String, "abcde", needs to be stored. The second String may be stored as (1,3) - the offset and length of "bcd" in "abcde". If a number of Strings are identical, at least one copy must be stored, while the rest may be stored as an offset and length. For our purposes, storing a String in memory requires a single byte for each character in the String, plus 1 extra byte to terminate the String (this is the case if they are stored with a null terminator). Storing an offset and length, on the other hand, requires exactly 12 bytes. Given these specs, and s, determine the fewest number of bytes required to store all of the Strings in memory. Keep in mind that it sometimes requires fewer bytes to store the actual String than it does to store an offset and length.

Notes

Constraints

  • s will contain between 1 and 50 elements, inclusive.
  • Each element of s will contain between 0 and 50 characters, inclusive.
  • Each character in s will be a lowercase letter ('a'-'z').
