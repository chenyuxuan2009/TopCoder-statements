Problem Statement for "CinemaProblem"

Problem Statement

The participants of a famous programming contest have gone to the cinema. The cinema hall contains several rows, each containing exactly seatsPerRow seats. The movie is very popular, so the contestants' assigned seats are scattered all over the cinema hall, and there are no vacant seats. Because the contest participants are very sociable with each other, they want to sit together according to the following rule. They must sit in such a way that every row containing a participant contains only participants. If the total number of participants is not evenly divisible by seatsPerRow, there can be one row that is not completely filled with participants.

You are given a int[] participantsInRow. The i-th element of participantsInRow is the number of contest participants sitting in the i-th row. Return the minimal number of swaps needed to satisfy the rule. Every person in the cinema hall is willing to swap seats with anybody else.

Definition

Class:
CinemaProblem
Method:
minSwaps
Parameters:
int, int[]
Returns:
int
Method signature:
int minSwaps(int seatsPerRow, int[] participantsInRow)
(be sure your method is public)

Constraints

  • seatsPerRow will be between 1 and 100, inclusive.
  • participantsInRow will contain between 1 and 50 elements, inclusive.
  • Each element of participantsInRow will be between 0 and seatsPerRow, inclusive.
  • participantsInRow will contain at least one positive element.
