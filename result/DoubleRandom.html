Problem Statement for "DoubleRandom"

Problem Statement

One particular type of random number generator is called a lagged Fibonacci generator. The nth random number generated is denoted F(n). Similar to the Fibonacci sequence, F is defined recursively as: 
     F( n ) = ( F( n-24 ) + F( n-55 ) ) mod m     for n >= 55 .
m must be defined before hand. A problem with such a generator is that you need 55 "setup values" to get the generator started. These 55 integers will be the values for F(0), F(1), ..., F(54). To obtain these setup values we will use a different kind of random number generator called a linear congruential generator. The kth random number generated by such a generator is denoted X(k). X is defined recursively as: 
     X( k ) = ( a*X( k-1 ) + c ) mod m     for k >= 1 .
m, a, c, and X(0) must be defined before hand.

In this problem you will be given m, a, and c as well as x0 which is X(0). You will use the linear congruential generator to produce 55 values for the lagged fibonacci generator. In other words F(0) = X(0), F(1) = X(1), ..., F(54) = X(54). You will then, using the same m value as in the linear congruential generator, compute and return F(n) for the given n.

Notes

  • In both formulas above, mod denotes the modulus operation written as % in many programming languages.

Constraints

  • a and c will be between 0 and 1000 inclusive.
  • m will be between 2 and 1000000 inclusive.
  • n will be between 0 and 1000000 inclusive.
  • x0 will be between 0 and m-1 inclusive.
