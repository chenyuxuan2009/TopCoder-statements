Problem Statement for "MovinAround"

Problem Statement

You are initially standing at xy-coordinate (xinit,yinit) facing in the positive y-direction. A list of commands are given to you. Each command will be one of (quotes for clarity):
  • "Turn Right" : Turn 90 degrees to the right (i.e. if up you will be facing right; if right you will be facing down...)
  • "Move Forward" : Move 1 unit in the direction you are facing.
Note, x-coordinates increase to the right, and y-coordinates increase upward. Each command should be followed one after the other. You will return the maximum distance from your initial location to the location you are at after any of the commands. Suppose you were at position (xcurr,ycurr) after executing one of the commands. The distance from this location to the inital location is given by:
	Sqrt( (xcurr-xinit)^2 + (ycurr-yinit)^2)
where Sqrt is the square root function, and ^ denotes exponentiation. To reiterate, you will return the maximum of any of these distances which are calculated after each command. See examples for further clarification.

Definition

Class:
MovinAround
Method:
distance
Parameters:
String[], int, int
Returns:
double
Method signature:
double distance(String[] moves, int xinit, int yinit)
(be sure your method is public)

Constraints

  • moves will contain between 1 and 50 elements inclusive
  • Each element of moves will be one of (quotes for clarity): "Move Forward", "Turn Right".
  • xinit will be between -100 and 100 inclusive
  • yinit will be between -100 and 100 inclusive
