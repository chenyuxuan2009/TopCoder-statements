Problem Statement

A store has been notified by a manufacturer that a certain product it carries has been deemed unsafe. The store must now send out recall notices to all customers who purchased that product. Customer information will be given as a String[], customers, where each element represents a single customer and is formatted as (quotes and brackets for clarity only): "<CUSTOMER_ID> <CUSTOMER_NAME>", where <CUSTOMER_ID> is an integer which uniquely identifies a customer, and <CUSTOMER_NAME> is the name of the customer. Transaction records will be given as a String[], transactions, where each element represents a single purchase and is formatted as (quotes and brackets for clarity only): "<CUSTOMER_ID> <PRODUCT_ID>", where <CUSTOMER_ID> is an integer which corresponds directly to the <CUSTOMER_ID>s in customers and <PRODUCT_ID> is an integer which identifies a unique product. Given an int, productId, return the names of all the customers who have purchased the recalled product identified by productId. The customer names should be returned as a String[] sorted in alphabetical order. If no customers meet the criteria, return an empty String[].

Constraints

  • transactions and customers will each contain between 1 and 50 elements, inclusive.
  • Each element of transactions will be formatted as " " where: - is an integer between 0 and 100, inclusive, with no extra leading zeros. - exists in customers - is an integer between 0 and 100, inclusive, with no extra leading zeros. - and are separated by a single space.
  • transactions contains no duplicate elements.
  • Each element of customers will contain between 3 and 50 characters, inclusive.
  • Each element of customers will be formatted as " " where: - is an integer between 0 and 100, inclusive, with no extra leading zeros. - contains only lowercase letters ('a'-'z') and is at least one letter long. - and are separated by a single space.
  • There are no duplicate s or s in customers.
  • productId is between 0 and 100, inclusive.
