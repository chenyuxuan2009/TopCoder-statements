Problem Statement for "BestCoder"

Problem Statement

There are a lot of programming contests in this world. Let's determine who is the BestCoder of this year! The BestCoder is the one who received the most winning points during the whole year. If there are multiple coders with the most winnning points, each of them will be regarded as the BestCoder. It is December 1 now, and you want to predict who can be the BestCoder.

There are N coders in question, numbered from 0 to N - 1. Coder j has received points[j] winning points by November.
There are M kinds of important contests, numbered from 0 to M - 1. For contest i, matches[i] matches will be held in December. In each match of contest i, the top wins[i] coders will receive 1 winning point and all other coders will not receive winning points. There will be no tie within each match.
The j-th character of participation[i] is 'Y' if coder j will participate in all matches of contest i, or 'N' if coder j will not participate in any matches of contest i. There are no other coders participating in these matches. That is, the number of participants in each match of contest i is equal to the number of 'Y's in participation[i].

Return a String containing exactly N characters, the j-th character of which is 'Y' if it is possible for coder j to be the BestCoder, or 'N' otherwise. Assume that any outcome is possible for each match that will be held in December. Note that all indices are 0-based.

Constraints

  • N, the number of elements in points, will be between 1 and 40, inclusive.
  • M, the number of elements in matches, will be between 1 and 40, inclusive.
  • Each element of points will be between 0 and 1,000,000,000, inclusive.
  • Each element of matches will be between 1 and 100,000, inclusive.
  • wins will contain exactly M elements.
  • participation will contain exactly M elements.
  • Each element of participation will contain exactly N characters.
  • Each character in participation will be either 'Y' or 'N'.
  • For each index i, wins[i] will be between 1 and the number of 'Y's in participation[i], inclusive.
