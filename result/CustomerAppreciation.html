Problem Statement

A store has decided to reward its best customers by sending them discount coupons in the mail. First, it must use its records to identify its highest spending customers. Customer information will be given as a String[], customers, where each element represents a single customer and is formatted as (quotes and brackets for clarity only): "<CUSTOMER_ID> <CUSTOMER_NAME>", where <CUSTOMER_ID> is an integer which uniquely identifies a customer, and <CUSTOMER_NAME> is the name of the customer. Transaction records will be given as a String[], transactions, where each element represents the total transactions for a single customer and is formatted as (quotes and brackets for clarity only): "<CUSTOMER_ID> <PURCHASE_AMOUNT>", where <CUSTOMER_ID> is an integer which corresponds directly to the <CUSTOMER_ID>s in customers, and <PURCHASE_AMOUNT> is the total amount of money spent by that customer in dollars. Given an int, threshold, return the names of all the customers who have spent at least threshold dollars. The customer names should be returned as a String[] sorted in alphabetical order. If no customers meet the criteria, return an empty String[].

Constraints

  • transactions and customers will each contain between 1 and 50 elements, inclusive.
  • Each element of transactions will be formatted as " " where: - is an integer between 0 and 100, inclusive, with no extra leading zeros. - exists in customers - is an integer between 1 and 99999, inclusive, with no extra leading zeros. - and are separated by a single space.
  • There are no duplicate s in transactions.
  • Each element of customers will contain between 3 and 50 characters, inclusive.
  • Each element of customers will be formatted as " " where: - is an integer between 0 and 100, inclusive, with no extra leading zeros. - contains only lowercase letters ('a'-'z') and is at least one letter long. - and are separated by a single space.
  • There are no duplicate s or s in customers.
  • threshold is between 1000 and 50000, inclusive.
