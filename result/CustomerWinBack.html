Problem Statement for "CustomerWinBack"

Problem Statement

A store has decided to embark on a campaign to win back former customers that were lost to its competitors. First, it must check its sales records to determine which customers have not made purchases in the longest time. Customer information will be given as a String[], customers, where each element represents a single customer and is formatted as (quotes and brackets for clarity only): "<CUSTOMER_ID> <CUSTOMER_NAME>", where <CUSTOMER_ID> is an integer which uniquely identifies a customer, and <CUSTOMER_NAME> is the name of the customer. Sales records will be given as a String[], sales, where each element represents the most recent purchase by a single customer and is formatted as (quotes and brackets for clarity only): "<CUSTOMER_ID> <DAYS_ELAPSED>", where <CUSTOMER_ID> is an integer which corresponds directly to the <CUSTOMER_ID>s in customers, and <DAYS_ELAPSED> is the number of days since the store opened that the purchase was made. Given an int, currentDay, and an int, threshold, return the names of all the customers whose last purchase was made at least threshold days ago. currentDay is the number of days since the store first opened. The customer names should be returned as a String[] sorted in alphabetical order. If no customers meet the criteria, return an empty String[].

Constraints

  • Each element of sales will be formatted as " " where: - is an integer between 0 and 100, inclusive, with no extra leading zeros. - exists in customers - is an integer between 1 and currentDay, inclusive, with no extra leading zeros. - and are separated by a single space.
  • There are no duplicate s in sales.
  • Each element of customers will contain between 3 and 50 characters, inclusive.
  • Each element of customers will be formatted as " " where: - is an integer between 0 and 100, inclusive, with no extra leading zeros. - contains only lowercase letters ('a'-'z') and is at least one letter long. - and are separated by a single space.
  • There are no duplicate s or s in customers.
  • currentDay is between 1000 and 10000, inclusive.
  • threshold is between 500 and 10000, inclusive.
