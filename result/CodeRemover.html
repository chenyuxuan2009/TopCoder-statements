Problem Statement for "CodeRemover"

Problem Statement

JavaServer Pages(JSP) provide a way to generate dynamic web pages for clients. In its simplest incarnation, a JSP allows the developer to mix static HTML with dynamic Java code which is run on the server. In this problem, your task is to build a simplified parser that will split a JSP into two parts - HTML and Java. Your method should then return the HTML part only.

For the purposes of this problem, we will only consider one type of embedded Java code. This code will be set off from the HTML portions of the JSP by <% and %>. The former will indicate the opening of a block of code, and the latter will indicate the closing of a block of code.

Your task is to find all of the blocks of code within a JSP, which will be given to you as a String[], each element of which represents a single line. You should remove all of the code sections, starting with <% and ending with %>. If this generates any empty lines, you should remove those, but you should never concatenate lines (see example 1). A line containing only spaces is not considered empty. Your method should return the result as a String[], with the code removed.

Notes

  • Sections of code may span multiple lines.
  • In order for <% to open a block of code, both characters must be on the same line. Hence, if < is at the end of one line, and % is at the beginning of the next line, that does not open a block of code. The same thing goes for %>

Constraints

  • jsp will contain between 1 and 50 elements, inclusive.
  • Each element of jsp will contain between 0 and 50 characters, inclusive.
  • Each character of jsp will have ASCII value between 32 and 126, inclusive.
  • jsp will be well formed, meaning that every <% has a corresponding %>.
