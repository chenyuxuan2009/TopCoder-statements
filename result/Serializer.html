Problem Statement for "Serializer"

Problem Statement

When transferring data across a network, the data must be serialized as a sequence of bytes. In this problem, we want to write a custom serialization for a String[], which is easily readable to developers who might want to sniff packets for debugging purposes. Your task is to take a String[], and return a serialized version of it as a String according to the following rules.
1) You should use a '{' to mark the beginning of the serialization and a '}' to mark the end of the serialization.
2) Each element should be surrounded by a single set of double quotes.
3) All of the elements should then be concatenated with a single comma between each pair of adjacent elements.
Thus, the serialized version of the String[] will end up looking almost identical to the way it would appear in the source code of Java, C++, or C#. So, if you had a String with 2 elements: &quot;hello&quot; and &quot;world&quot; (quotes for clarity only), you would return the string:
{&quot;hello&quot;,&quot;world&quot;}

Definition

Class:
Serializer
Method:

Parameters:

Returns:

Method signature:
String serializeStringArray(String[] array)
(be sure your method is public)

Notes

  • In reality, you would have to worry about having double quotes within strings, and would have to use escape characters. However, in this problem you needn't worry about this because neither backslashes ('\'), nor double quotes ('&quot;') will appear in the input.
  • You would also have to worry about the String[] being null, and elements of the String[] being null, but in this problem, neither the String[], nor elements of the String[] will be null.
  • Note that the expected results displayed in the examples have double quotes around them and backslashes before double quotes within the return. This is for clarity only. The sequence \&quot; means a single double quote, and the leading and trailing double quotes serve to signify the beginning and end of the result string.

Constraints

  • array will contain between 0 and 50 elements, inclusive.
  • Each element of array will contain between 0 and 50 characters, inclusive.
  • All characters in each element of array will be printable ASCII characters. In other words, they will have ASCII values between 32 and 126, inclusive (all of these characters can appear on a standard keyboard). The only exceptions to this rule are the backslash ('\') and the double quote ('"'), which will not be allowed.
