Problem Statement for "WolfOnlineJudge"

Problem Statement

Wolf Sothe has made an online judge website called Wolf Online Judge. He came up with a special scoring system.

In this scoring system, the score value of a problem depends on how many coders solve the problem. This value can be calculated as 64.0/(the number of coders who solved the problem) if at least one coder solved it, and 0 otherwise. The total score of each coder is the sum of score values of all problems that the coder solved.

You are given a String[] solved with N elements, where N is the number of coders. Each element of solved contains M characters, where M is the number of problems. If the j-th character of the i-th element of solved is 'Y', the i-th coder solved the j-th problem; if it's 'N' instead, the i-th coder didn't solve the j-th problem.

Compute and return a double[] with N elements. The i-th element of this double[] is equal to the i-th coder's total score.

Definition

Class:
WolfOnlineJudge
Method:
score
Parameters:
String[]
Returns:
double[]
Method signature:
double[] score(String[] solved)
(be sure your method is public)

Notes

  • Your return value must have absolute or relative error smaller than 1e-9.

Constraints

  • The number of elements in solved will be between 1 and 50, inclusive.
  • Each element in solved will have the same number of characters.
  • The length of each element in solved will be between 1 and 50, inclusive.
  • Every character will be either 'Y' or 'N'.
