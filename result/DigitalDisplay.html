Problem Statement for "DigitalDisplay"

Problem Statement

A digital clock displays time in the format &quot;DD:DD:DD&quot; (quotes for clarity only), where each D is a single digit, and each pair of contiguous digits represents either the hour, minute, or second (each of these three units will occur exactly once). Unfortunately, we do not know the ordering of the time units, so need to figure out the number of valid ways to interpret the displayed time.

The hour must be between 01 and 12, inclusive, the minute must be between 00 and 59, inclusive, and the second must be between 00 and 59, inclusive.
For example, &quot;21:23:01&quot; can be intepreted in two ways: &quot;minute:second:hour&quot; or &quot;second:minute:hour&quot;.

Definition

Class:
DigitalDisplay
Method:
waysToInterpret
Parameters:
String
Returns:
int
Method signature:
int waysToInterpret(String display)
(be sure your method is public)

Constraints

  • display will contain exactly 8 characters.
  • display will be formatted as "DD:DD:DD" (quotes for clarity only), where each D is a digit ('0'-'9').

Examples

    Returns: 2
