Problem Statement

This problem wasn't originally going to be part of this contest.
Unfortunately, the problem setter made a fatal mistake in the problem that was supposed to be used originally - when creating the input for that problem, he accidentally erased some numbers from it.
Therefore, we decided to replace that problem with this one.


The input of the original problem consisted of a int[][] T, the elements of which are preceded by their sizes.
All elements of input are between 0 and 500, inclusive. For the purpose of this problem, we'll view it as a single int[] input.


Specifically, a valid input without erased numbers must look as follows:

  • First, L0=input[0].
  • Then, L0 integers - input[1] through input[L0]. This corresponds to the int[] T[0].
  • Then, L1=input[L0+1].
  • Then, L1 integers - input[L0+2] through input[L0+L1+1]. This corresponds to the int[] T[1].
  • Then, L2=input[L0+L1+2], etc. till the end of the input.
There must not be any integers left over at the end of the input.


For example, if T={{1,2,3},{3,5}}, then input={3,1,2,3,2,3,5}, and if T={{0,0,0},{},{4,3,2,1}}, then input={3,0,0,0,0,4,4,3,2,1}.


Some elements of input are deleted. A deleted element is represented by number -1.
Reconstructing the input means replacing all -1s in input by non-negative integers in such a way, that the resulting int[] is a valid input.
(In particular, all the elements of input that correspond to elements of int[]s T[i] must be between 0 and 500, inclusive.)


Compute and return the number of ways to reconstruct the input, modulo 1,000,000,007.
