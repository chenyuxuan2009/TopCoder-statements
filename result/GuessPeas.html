Problem Statement for "GuessPeas"

Problem Statement

John has recently held a "guess the number of peas in the jar" competition, and now he needs to find out who the winner is. He has recorded all the guesses each contestant has made, as well as their names. The contestant who is closest to the real answer wins. You may assume that their is a unique winner.

Create a class GuessPeas containing the method winner which takes as input a int[] guess and a String[] name and an int peas, the number of peas in the glass jar, and returns a String, the name of the contestant with the best guess. Element i in guess and name correspond to contestant i.

Definition

Class:
GuessPeas
Method:
winner
Parameters:
int[], String[], int
Returns:
String
Method signature:
String winner(int[] guess, String[] name, int peas)
(be sure your method is public)

Constraints

  • guess will contain between 1 and 50 elements.
  • name will contain between 1 and 50 elements.
  • guess will contain the same number of elements as name.
  • Each element in guess will be between 1 and 10000.
  • Each element in name will contain between 1 and 50 characters, inclusive.
  • Each element in name will only contain upper and lower case letters ('A'-'Z','a'-'z').
  • peas will be between 1 and 10000.
  • There will be exactly one guess which is the best one.
