Problem Statement for "EllysPyramid"

Problem Statement

Elly plays a computer game where she must build Aztec-like pyramid. The game is two-dimensional and the ground is represented as the X-axis.

Large stones of height 1 pixel and various widths fall from the sky at different locations (measured from the left side of the screen). The stones fall one after another and the girl can decide for each of them to either destroy it or leave it fall. The first stone block the girl decides to leave falls on the ground - that is the base of the pyramid. After that whenever she decides to leave a block fall, it must fall completely within the interval of the last fallen block. For example, if the last fallen block started at pixel L and ended at pixel R (both inclusive), the next non-destroyed block should have start and end within the interval [L, R].

By leaving stone blocks fall on top of previously fallen blocks a pyramid is build. The goal of the game is to build the highest possible pyramid.

You are given two int[]s L and R. The i-th element of these describes the left and right inclusive boundaries of the i-th stone block. Return an int giving the maximal height of a pyramid the girl can build.

Definition

Class:
EllysPyramid
Method:
getHeight
Parameters:
int[], int[]
Returns:
int
Method signature:
int getHeight(int[] L, int[] R)
(be sure your method is public)

Notes

  • Please note that the girl cannot move the blocks - she can only decide whether to destroy them or let them fall.
  • Note that only one block can touch the ground and each non-destroyed block after that must fall within the boundaries of the top block of the pyramid.
  • It is possible that a stone starting at L and ending at R falls on a stone that also starts at L and ends at R (i.e. covers it exactly).

Constraints

  • L and R will contain between 1 and 50 elements, inclusive.
  • L and R will contain the same number of elements.
  • Each element of L and R will be between 1 and 1000, inclusive.
  • For each valid index i, L[i] will be less than or equal to R[i].

    {4, 7, 7, 6, 7, 9, 5, 5}

    Returns: 5

    It is better to destroy the first block (since it is rather small) and let the base of the pyramid be the second one - from 2 to 7. The third one we can also leave fall - [3, 7] is fully within [2, 7]. We also take [4, 6]. The next block (from 2 to 7) cannot be taken if the last one is [4, 6] (it is outside its boundaries). Please note that if we've discarded [3, 7] and [4, 6] we could have taken [2, 7] (on top of the other [2, 7]). However, this would lead to a sub-optimal answer. We cannot take [3, 9] either, since it is also outside the interval [4, 6]. Finally, we take both [5, 5] blocks, for a total height of 5: ([2, 7], [3, 7], [4, 6], [5, 5], [5, 5]).
