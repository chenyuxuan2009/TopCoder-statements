Problem Statement for "RoundCountDrawing"

Problem Statement

A group of n people are getting ready to play a game, and they will pick a starter using the following method: The n players stand in a circle and are numbered from 0 to n-1 in clockwise order (it doesn't matter where the numbering starts). Then, each player secretly and randomly picks a number between 0 and k, inclusive. Finally, all the chosen numbers are added up to get a number R. Starting at player 0 and going in clockwise order, the players will count from 0 to R. Whoever gets R will be the starter.

You will be given n, k and i and have to return the probability that the participant with number i is picked as starter, assuming each participant chooses his number independently and uniformly between 0 and k, inclusive.

Definition

Class:
RoundCountDrawing
Method:
getProb
Parameters:
int, int, int
Returns:
double
Method signature:
double getProb(int n, int k, int i)
(be sure your method is public)

Notes

  • Each player chooses his number in a purely random way, without employing any kind of strategy.
  • The returned value must be accurate to within a relative or absolute value of 1E-9.

Constraints

  • n will be between 1 and 1000, inclusive.
  • k will be between 1 and 1000000000 (109), inclusive.
  • i will be between 0 and n-1, inclusive.
