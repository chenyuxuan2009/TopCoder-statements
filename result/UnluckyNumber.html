Problem Statement for "UnluckyNumber"

Problem Statement

You are given a simple undirected graph consisting of n vertices and m edges. The vertices are numbered 1, 2, ..., n. You are also given an unlucky number z.

We need to colour each vertex of the graph with one of k colours. The colours are numbered 1, 2, ..., k.

Let c[i] be the color assigned to vertex i. A coloring is valid if for each edge (i, j) the value c[i] + c[j] does not equal the unlucky number z.

Find the number of such colourings modulo 109 + 7.


You are given:

  • The number of vertices and edges: n and m.
  • The number of colours and the unlucky number that should not be the sum of colours for any edge: k and z.
  • The list of edges: two int[]s x and y with m elements each. For each valid index i, there is an edge connecting x[i] and y[i].

Definition

Class:
UnluckyNumber
Method:
numberOfColourings
Parameters:
int, int, int, int, int[], int[]
Returns:
int
Method signature:
int numberOfColourings(int n, int m, int k, int z, int[] x, int[] y)
(be sure your method is public)

Constraints

  • 1 <= n <= 18
  • 0 <= m <= min(18, n*(n - 1)/2)
  • 1 <= k <= 10^9
  • 2 <= z <= 2*k
  • x and y will each contain m elements
  • 1 <= x[i], y[i] <= n
  • There will be no self-loops and all edges will be distinct.
