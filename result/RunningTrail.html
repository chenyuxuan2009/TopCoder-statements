Problem Statement for "RunningTrail"

Problem Statement

I like to run. When I move to a new area, I want to explore the surroundings and locate a good running trail. There are many paths and junctions I can choose to be part of my running trail, but I want the total length of the trail to be as close to a specific length as possible (but always greater than zero). Given a set of paths as a String[], determine a trail consisting of at least one path whose length is as close to the desired length as possible. While I don't mind running the same path several times (see example 0), I don't want to run to a junction at the end of a path, and then turn around and run back to the previous junction along the same path (see example 1).

Each path will be described by a String in the format (quotes for clarity only) "<junctionA> <junctionB> <length>", where junctionA, junctionB and length are all positive integers. Several paths might go between the same two junctions, and I can run along a path in both directions. The trail should always start and end at junction 1. Create a class RunningTrail containing the method bestTrail which takes a String[] paths and an int desiredLength, and returns an int containing the length of the trail closest to the desired length. In case there are several trails equally close to the desired length, return the length of the shortest of them.

RunningTrail

Constraints

  • paths will contain between 2 and 50 elements, inclusive.
  • desiredLength will be between 1 and 100000, inclusive.
  • Each element in paths will be in the format above, with no leading zeros or extra spaces.
  • In each element of paths, and will be integers between 1 and 50, inclusive.
  • In each element of paths, and will be distinct.
  • In each element of paths, will be an integer between 1 and 5000, inclusive.
  • All paths in paths will be reachable from junction 1.
  • At least one trail of positive length will exist that starts and ends at junction 1.
