Problem Statement for "KNN"

Problem Statement

Classification problems are quite common in many areas. One common example is in diagnosing diseases where the symptoms of some illness must be classified as a particular disease so that it can be treated properly. Other examples abound, but the one thing that they have in common is that there are a number of known variables, and the goal is to learn some function from the known variables to the classifications.

A common way to deal with this problem is to start with some relatively large set of cases for the problem, whose classifications have already been determined by some independent method. Then, when a new case arises, whose classification is unknown, one can look for similar cases with known classifications, and assume that if the values of the variables are similar, then the final classification will be too. One relatively simple algorithm that often does well at this problem is called K-Nearest Neighbor (KNN). In this algorithm a distance metric is established for evaluating the distance between two cases in the problem. When a new case comes up, the distances from that case to all of the cases with known classifications are calculated. Then, the K cases with the lowest distances are chosen, and the classification that is most common among those cases is deemed to be the classification for the new case.

In order to make this method most effective, the distance metric must be well suited for the problem. Constraints

  • known will contain between 2 and 50 elements, inclusive.
  • K will be between 1 and the number of elements in known - 1, inclusive.
  • unknown will contain between 1 and 50 elements, inclusive.
  • Each element of known will be formatted as " ... ", where each will be an integer between -1000 and 1000, inclusive, with no extra leading zeros, and will contain between 1 and 10 uppercase letters ('A'-'Z'), inclusive.
  • Each element of known will contain the same number of variables as unknown.
  • Each variable in each element of known will be between -1000 and 1000, inclusive.
  • Each element of unknown will be between -1000 and 1000, inclusive.
  • weights will contain the same number of elements as unknown.
  • Each element of weights will be between 0 and 10, inclusive.
  • There will be exactly one classification that will be the most common in the K nearest cases.
  • The (K+1)th nearest case will not be the exact same distance from the unknown case as the Kth nearest case, so there will be K cases that are closer than all of the other cases.
